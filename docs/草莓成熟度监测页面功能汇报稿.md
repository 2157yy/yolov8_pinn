> 汇报日期：2026年6月24日

## 一、项目背景与目标

本项目面向智能温室草莓种植场景，构建一套基于 **YOLOv8 深度学习模型**的草莓成熟度自动检测与可视化监控系统。系统核心目标包括：

- 实现草莓果实成熟度的**自动识别与三分类评估**（半熟 / 成熟 / 未熟）
- 提供**直观的可视化检测结果**，包含标注框图和统计分布
- 与温室环境监控台联动，形成"感知—诊断—决策"一体化智能农业闭环

#### 3.1.1 图片上传模块

| 功能 | 说明 |
|------|------|
| 点击上传 | 点击上传区域触发文件选择器 |
| 拖拽上传 | 支持从文件管理器直接拖拽图片到上传区 |
| 格式校验 | 仅允许 JPG / PNG / BMP / TIFF / WebP 格式 |

| 移除重选 | 提供"移除图片"按钮，可随时更换上传内容 |



#### 3.1.3 检测结果展示

**（1）成熟度统计面板**

以三列卡片形式展示各类别统计数据：

| 类别 | 中文名称 | 颜色标识 | 展示内容 |
|------|---------|---------|---------|
| halfripe | 半熟 | 黄色 `#ecc94b` | 数量 + 百分比 + 进度条 |
| ripe | 成熟 | 绿色 `#48bb78` | 数量 + 百分比 + 进度条 |
| unripe | 未熟 | 橙色 `#ed8936` | 数量 + 百分比 + 进度条 |

进度条宽度根据该类占总体百分比动态渲染，带有 0.6s CSS 过渡动画。

**（2）检测详情列表**

逐条展示每个检测目标的：
- 类别标签（彩色胶囊样式 Chip）
- 置信度百分比（保留一位小数）
- 边界框坐标 `[x1, y1, x2, y2]`

**（3）标注结果图像**

- 在原图上叠加彩色边界框：
  - 半熟（halfripe）→ 黄色边框
  - 成熟（ripe）→ 绿色边框
  - 未熟（unripe）→ 橙色边框
- 每个框标注类别名称与置信度分数
- 图像以 Base64 编码从后端直接返回，无需额外文件请求
- OpenCV 绘图参数：线宽 2px，标签字体 `FONT_HERSHEY_SIMPLEX` 0.6

#### 3.1.4 交互与导航

- 顶部实时时钟（每秒更新，中文格式）
- 状态指示胶囊：`等待上传` / `检测中...` / `检测完成`
- "跳转至主监控台"按钮，一键返回 Dashboard

### 3.2 主监控台草莓分析集成（`/dashboard`）

**页面路径**：`smart-greenhouse-dashboard/src/views/Dashboard.vue`

主监控台在原有温室环境监控基础上，深度集成了草莓成熟度分析能力：

| 功能模块 | 说明 |
|---------|------|
| 环境指标监控 | 温度、湿度、光照、CO2、土壤湿度、pH 值，实时模拟刷新 |
| 温度趋势图 | 日 / 周 / 月三档切换，柱状图可视化 |
| 实时温度仪表 | 环形仪表盘动态展示当前温度 |
| 设备控制 | 通风、灌溉、补光、加湿四路开关控制 |
| 告警通知 | 传感器阈值告警（如湿度偏低） |
| **Qwen 草莓咨询助手** | 图片上传 + 成熟度检测 + 大模型流式分析 |
| 快捷跳转 | "跳转至成熟度检测"按钮 |

**Qwen 草莓咨询助手工作流**：
1. 用户上传草莓图片并提出问题（或选择快捷问题）
2. 系统执行 YOLO 成熟度检测
3. 检测结果注入 Qwen 系统提示词
4. SSE 流式返回分析结果（三阶段：YOLO 检测 → 分析推理 → 最终回复）
5. 结果卡片展示检测到的类别与置信度，绿色（>70%）/ 橙色（≤70%）

### 3.3 成熟度数据集管理页面（`/datasets`）

**页面路径**：`smart-greenhouse-dashboard/src/views/DatasetManager.vue`

| 功能模块 | 说明 |
|---------|------|
| 存储概览 | 数据库模式（MySQL / 内存）、数据集总数、样本总数、当前启用数据集 ID |
| 数据集导入 | 选择本地 YOLO 格式文件夹，配置类名映射与别名，一键导入 |
| 预设管理 | 内置公开数据集预设（如 `amitamola/Strawberry-Counting-and-Ripeness-detection`），一键套用 |
| 数据集注册表 | 已导入数据集列表，展示样本数、标注数、类分布、导入时间 |
| 样本预览 | 网格展示数据集样本缩略图、标注信息、数据划分（train/val/test） |
| 启用切换 | 将指定数据集设为当前激活状态 |

**类名统一映射机制**：不同公开数据集的类名（如 `partially_ripe`、`fully_ripe`）通过别名映射统一为标准三类（halfripe / ripe / unripe），确保多源数据的互操作性。

## 四、技术架构详述

### 4.1 前端技术栈

| 技术 | 版本/说明 |
|------|----------|
| Vue 3 | Composition API + Options API 混合模式 |
| Vue Router | Hash 模式路由（3 个页面 + 默认重定向） |
| 样式方案 | Scoped CSS + 暗色主题 + 毛玻璃效果 |
| 图表 | 自建柱状图 + 环形仪表（纯 CSS） |
| 流式通信 | SSE（EventSource）接收 Qwen 分析流 |
| 构建工具 | Vite |

### 4.2 后端技术栈

| 技术 | 说明 |
|------|------|
| Node.js / Express | API 服务 + 静态文件托管（端口 3000） |
| multer | 文件上传中间件（16MB 图片 / 64MB 数据集） |
| Python 子进程 | `spawn` 方式调用桥接脚本，stdin/stdout JSON 通信 |
| 超时机制 | 检测 120s / Qwen 对话 180s |
| MySQL | 成熟度数据集持久化存储（通过 `maturityDatasetStore.js`） |

### 4.3 AI 模型与算法

| 组件 | 说明 |
|------|------|
| 检测模型 | YOLO（Ultralytics），权重 `strawberry-maturity-s-best.pt`（22.5 MB） |
| 模型来源 | 公开预训练模型（GitHub: `tahsinsoyak/strawberry-ai-on-web`） |
| 分类体系 | 3 类：halfripe（半熟）、ripe（成熟）、unripe（未熟） |
| 置信度阈值 | 默认 0.25 |
| 标注绘制 | OpenCV（cv2）：矩形框 + 文字标签 |
| 大语言模型 | Qwen3.6-plus（阿里 DashScope API，OpenAI 兼容协议） |
| 感知模块 | `yolo_adapter.py` — 7 类草莓病害检测适配器 |
| 诊断 Agent | 规则版 + Qwen 版双模式（评估置信度、光照影响、历史一致性） |
| 决策 Agent | 规则版 + Qwen 版双模式（采摘阈值 0.80、补光阈值 0.30） |
| 记忆 Agent | JSONL 文件存储（`JsonlMemoryAgent`）+ 内存版（`InMemoryMemoryAgent`） |

## 六、功能亮点总结

| 亮点 | 说明 |
|------|------|
| **端到端闭环** | 图片上传 → YOLO 检测 → 标注可视化 → Qwen 分析 → 决策建议，全流程打通 |
| **三页联动** | 监控台（Dashboard）→ 成熟度检测（Maturity）→ 数据集管理（DatasetManager），无缝跳转 |
| **双语言支持** | 界面中文显示，代码注释英文字段，API 字段英文标准 |
| **多源数据统一** | 类名别名映射机制，兼容不同公开数据集的标注规范 |
| **流式体验** | Qwen 分析使用 SSE 流式推送，实时展示推理进度（YOLO 检测 → 分析 → 回复） |
| **优雅降级** | Python 解释器多候选回退（`python3` → `python`），检测超时自动终止 |
| **安全设计** | 数据集文件读取路径遍历防护（`guardPrefix` 校验），上传格式白名单 |
| **模型可替换** | 模型路径、置信度阈值均可通过参数配置，支持未来模型升级 |
